# 📌 Find the Difference

---

# 📝 Problem Statement

You are given two strings `s` and `t`.
- String `t` is generated by **randomly shuffling** string `s` and then **adding one more letter** at a random position.
- Return the letter that was **added** to `t`.

### **Constraints**
- `0 <= s.length <= 1000`
- `t.length == s.length + 1`
- `s` and `t` consist of lowercase English letters.

### **Examples**
| Input | Output | Explanation |
|-------|--------|-------------|
| `s = "abcd"`, `t = "abcde"` | `"e"` | `'e'` is the letter that was added. |
| `s = ""`, `t = "y"` | `"y"` | `t` contains only one character. |

---

# 💡 Intuition

The problem requires identifying the extra character added to string `t` compared to string `s`. Since `t` is a shuffled version of `s` with one extra character, we can leverage the fact that:
- The extra character will **disrupt the frequency balance** between `s` and `t`.
- If we **count the frequency** of each character in both strings, the extra character will have a **frequency mismatch**.
- Alternatively, we can use **bitwise XOR** to cancel out all matching characters, leaving only the extra character.

This leads to two natural approaches:
1. **Frequency Counting** (Brute Force)
2. **Bitwise XOR** (Optimal)

---

# 🐌 Brute Force Approach

## 🔹 Approach

We can solve this problem by:
1. Counting the frequency of each character in `s`.
2. Counting the frequency of each character in `t`.
3. Comparing the two frequency maps.
4. The character with a **higher count in `t`** is the added character.

This approach is straightforward but requires **O(n) space** to store frequency counts.

---

## 🔹 Algorithm

1. Initialize a frequency array of size 26 (for lowercase English letters).
2. Traverse string `s` and increment the count for each character.
3. Traverse string `t` and decrement the count for each character.
4. The character with a **negative count** in the frequency array is the added character.

---

## 🔹 Code

```java
class Solution {
    public char findTheDifference(String s, String t) {
        int[] freq = new int[26];

        // Increment frequency for characters in s
        for (char c : s.toCharArray()) {
            freq[c - 'a']++;
        }

        // Decrement frequency for characters in t
        for (char c : t.toCharArray()) {
            freq[c - 'a']--;
        }

        // Find the character with negative frequency
        for (int i = 0; i < 26; i++) {
            if (freq[i] < 0) {
                return (char) (i + 'a');
            }
        }

        return ' '; // Should not reach here per problem constraints
    }
}
```

# ⚡ Optimal Approach

## 🔹 Approach

We can optimize space by using **bitwise XOR**:
- XOR all characters in `s` and `t`.
- Since XOR cancels out identical characters, the remaining value is the added character.

This approach uses **O(1) space** and is more efficient.

---

## 🔹 Why This Works

- XOR is **commutative and associative**, so order doesn’t matter.
- XOR of a character with itself is `0`.
- XOR of a character with `0` is the character itself.
- Thus, all matching characters cancel out, leaving only the extra character.

---

## 🔹 Algorithm

1. Initialize `result = 0`.
2. XOR all characters in `s` with `result`.
3. XOR all characters in `t` with `result`.
4. The final value of `result` is the ASCII value of the added character.

---

## 🔹 Code

```java
class Solution {
    public char findTheDifference(String s, String t) {
        char result = 0;

        for (char c : s.toCharArray()) {
            result ^= c;
        }

        for (char c : t.toCharArray()) {
            result ^= c;
        }

        return result;
    }
}
```

---

## 🔹 Detailed Dry Run

**Input:** `s = "abcd"`, `t = "abcde"`

| Step | Character | XOR Operation | `result` (ASCII) |
|------|-----------|---------------|------------------|
| 1    | `'a'`     | `0 ^ 'a'`     | `97`             |
| 2    | `'b'`     | `97 ^ 'b'`    | `33`             |
| 3    | `'c'`     | `33 ^ 'c'`    | `100`            |
| 4    | `'d'`     | `100 ^ 'd'`   | `0`              |
| 5    | `'a'`     | `0 ^ 'a'`     | `97`             |
| 6    | `'b'`     | `97 ^ 'b'`    | `33`             |
| 7    | `'c'`     | `33 ^ 'c'`    | `100`            |
| 8    | `'d'`     | `100 ^ 'd'`   | `0`              |
| 9    | `'e'`     | `0 ^ 'e'`     | `101`            |

**Final `result`:** `101` → `'e'`

# 🔍 Edge Cases

| Case | Input (`s`, `t`) | Expected Output | Explanation |
|------|------------------|-----------------|-------------|
| Empty `s` | `("", "y")` | `"y"` | Only one character in `t`. |
| Single Character | `("a", "aa")` | `"a"` | Duplicate character added. |
| All Same Characters | `("aaa", "aaaa")` | `"a"` | Extra `'a'` added. |
| Large Input | `s = "a".repeat(1000)`, `t = "a".repeat(1001)` | `"a"` | Handles upper constraint. |

---

# 📚 Key Takeaways

✅ **Frequency Counting** is intuitive but uses extra space.
✅ **Bitwise XOR** is optimal for space and time.
✅ XOR cancels out duplicates, leaving only the extra character.
✅ Both approaches run in **O(n) time**, but XOR is **O(1) space**.

---

# 🚀 Interview Tips

- **Follow-up:** What if `t` has **two extra characters**? (Answer: XOR won’t work; use frequency counting.)
- **Pitfall:** Forgetting to handle empty `s`.
- **Alternative:** Summing ASCII values (but may overflow for large inputs).
- **Optimization Insight:** XOR is **faster** and **space-efficient**.

---

# ✅ Conclusion

The **optimal solution** uses **bitwise XOR** to efficiently find the added character in **O(n) time and O(1) space**. This approach is **clean, fast, and interview-friendly**, making it the preferred choice for FAANG-level coding interviews.

**Key Insight:** XOR cancels out duplicates, leaving only the extra character. This is a powerful pattern for problems involving **single mismatches** in otherwise identical data.
